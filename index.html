A personal agent whose eval is your life, not your engagement.

K runs on your machine, holds your life's record locally, and stages recommendations a human decides on. It measures itself against one thing: did attention go where you meant it to go, did decisions get acted on, did open loops resolve. A pleasant system that never resolves anything is a failing system.

sovereign · advisory-first · silence-default · agpl-3.0 · you are the server

principles

1
sovereignty is structuralthe life record lives in local JSON; sensitive reasoning routes only to model lanes you control; there is no telemetry; the data directory is uncommittable by construction.
2
attention economicsevery interruption has a class (ambient / peripheral / focal), every category has a daily budget, one slot arbitrates what may speak. over-budget insight waits.
3
advisory-first autonomyacting requires an explicit grant; everything else is a staged recommendation with a named undo.
4
silence-defaultnothing renders unless it changes what you do next. empty states say what silence means.
5
truth first, anti-glazeno praise, no mirroring for agreement. missing data is named, never imputed. every rate carries its denominator.
6
consent-first sensesevery data source starts OFF; unconfigured is a clean state. ingestion is dedup-idempotent and fail-soft.
7
a governed factoryK builds itself through plans, units, and lanes — but protected files integrate only through reviewed paths, and genuine decisions become cards for the human.
8
recognition over recallessence inline, depth one gesture away.
9
honest instrumentationthe eval never fills missing data, never penalizes silence, and shows its own vitals to the human it measures for.

governance physics

The build runner is autonomous until it isn't: a safety floor (protected files integrate only after human-reviewed diffs), scope-drift checks (a unit touching files outside its declared scope halts), decision cards (genuine choices go to the human, one tap each), and a deploy gate (daemon-affecting changes ship through an intent/outcome handshake with rollback).

publishingthis repo is a fresh-history mirror of the private working repo — allowlist export through fail-closed gates (secret scan, identity lint, image ban)

how to use

1 · run the daemon

git clone https://github.com/withdhyan/k-daemon && cd k-daemon
npm install
cp .env.example .env.local        # choose your model lane(s)
node daemon/server.mjs            # binds 127.0.0.1:3003

2 · choose your sovereign lane

# .env.local — pick ONE
OPENROUTER_API_KEY=...            # ZDR-only routing, data_collection: deny
K_SOVEREIGN_PROVIDER=claude-cli   # or: pipe through a local CLI subscription
K_MIND_MODEL_PROVIDER=ollama      # or: fully local for background synthesis

the sovereign lane is where anything personal reasons. the optional frontier lane only ever sees turns the classifier deems non-sensitive — and never your substrate.

3 · connect senses (each one is opt-in)

WHOOP_CLIENT_ID=...               # developer.whoop.com → create app
WHOOP_CLIENT_SECRET=...
WHOOP_REDIRECT_URI=http://localhost:3003/api/whoop/callback
# then visit /api/whoop/connect once and approve

4 · pair the app

Build k-app (SwiftUI, iPad + iPhone) in Xcode, point cskBaseURL at your daemon, and the five surfaces — cadence, chat, build, mind, admin — come alive against your own record.

5 · let it earn autonomy

K starts advisory-everything. Grants are per-tool; the factory's floor and gates stay on regardless. The right posture is the one it ships with: it asks before it acts, and it is silent unless speaking changes your next move.

voice

K speaks like a quiet peer who did the work: plain prose, second person, terse, no exclamation marks, numbers rounded to what a human can use. Pipeline scaffolding — ids, template keys, raw floats — stays in payloads, never in copy.
